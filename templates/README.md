# PROJECT_NAME

Databricks project built with [Claude Code](https://docs.anthropic.com/en/docs/claude-code) + Databricks Asset Bundles.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Compute | Serverless |
| Storage | Unity Catalog (`PROJECT_CATALOG`) |
| Pipelines | Spark Declarative Pipelines (SDP) |
| Orchestration | Databricks Jobs / DABs |
| CI/CD | GitHub Actions → `databricks bundle deploy` |

## Schemas

| Schema | Purpose |
|--------|---------|
| `PROJECT_CATALOG.raw` | Bronze — source data as-is, append-only |
| `PROJECT_CATALOG.cleansed` | Silver — cleaned, deduplicated, typed |
| `PROJECT_CATALOG.curated` | Gold — business aggregations, metrics |

## Getting Started

```bash
# 1. Clone
git clone <repo-url> && cd PROJECT_NAME

# 2. Set up Python environment
uv venv && source .venv/bin/activate && uv pip install -r requirements.txt

# 3. Verify Databricks auth
databricks auth status

# 4. Create UC schemas (first time only)
databricks bundle run setup_infrastructure --target dev

# 5. Deploy
databricks bundle deploy --target dev
```

## Project Structure

```
PROJECT_NAME/
├── .claude/              # Claude Code config (commands, agents, settings)
├── .github/              # CI/CD, issue templates, CODEOWNERS
├── resources/            # DAB resource definitions (jobs, pipelines)
├── scripts/              # Infrastructure setup scripts
├── src/                  # Application code
├── databricks.yml        # Databricks Asset Bundle config
└── PLAN.md               # Project plan (generated by /plan)
```

## Commands

| Command | Description |
|---------|-------------|
| `/plan <use case>` | Break down a use case into PLAN.md + GitHub issues |
| `/ship [title]` | Test → commit → push → PR |
| `/demo-prep [topic]` | Prepare code for a customer demo |
| `/techdebt [area]` | Analyze code for improvements |

## Deployment

Push to `main` triggers automatic deployment via GitHub Actions:
```
main → databricks bundle deploy --target prod
```

Feature branches deploy to dev:
```
databricks bundle deploy --target dev
```
